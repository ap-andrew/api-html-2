Get Custom Report

	    
78

		Detail

	

    

		Enables the generation of a custom report that incorporates dimensions, measures, and filters.
		  
	

	

		A report in Analytics is made up of dimensions, measures and filters.

"Dimensions" enable you view data from various perspectives. For example, the "time" dimension show you the breakdown of calls for the last month, day or hour. The "throughput" dimension enables you to see which APIs are the most active, based on the amount of traffic, the raw throughput, you APs are experiencing. The "error rate" dimension enables you you pinpoint the APIs that are causing issues for your APi consumers--high error rates lead to frustrated app end users. The "average endpoint response time" dimension enables you see how your server infrastructure is performing, since it measures the length of time it takes a target to respond to Apigee.  By combining dimensions and measures you can generate reports that reveal customized perspectives on the performance and usage of your APIs.

16

		Detail

	

    

		Create an API product to enable developers to access a set of API resources (URIs)
		  
	

	

		Use the POST method to create a new API product. 
You need to:
Name the API product
Bundle Resources by defining resource paths (URIs)
Define the "Service Plan" settings for quota (permitted requests per day/month) and/or spike arrest (permitted requests per second/minute).  The service plan defines how much your developers can consume over time. At runtime, these variables are read by the Rate Limiting Policy applied to an API or Resource. Rate Limiting Variabes are set as API Product attributes:
developer.quota.limit - integer
developer.quota.interval - integer
developer.quota.timeunit second/minute/hour/day/month)
You can also use attributes to set any arbitrary metadata that you need for monitoring or analytics.
