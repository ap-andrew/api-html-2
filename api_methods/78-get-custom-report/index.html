Get Custom Report :: /environments/{environmentName}/stats/{customVariable}

API resource: Analytics

Verb: GET

Doc: A report in Analytics is made up of dimensions, measures and filters. &quot;Dimensions&quot; enable you view data from various perspectives. For example, the &quot;time&quot; dimension show you the breakdown of calls for the last month, day or hour. The &quot;throughput&quot; dimension enables you to see which APIs are the most active, based on the amount of traffic, the raw throughput, you APs are experiencing. The &quot;error rate&quot; dimension enables you you pinpoint the APIs that are causing issues for your APi consumers--high error rates lead to frustrated app end users. The &quot;average endpoint response time&quot; dimension enables you see how your server infrastructure is performing, since it measures the length of time it takes a target to respond to Apigee. By combining dimensions and measures you can generate reports that reveal customized perspectives on the performance and usage of your APIs.

Description: Enables the generation of a custom report that incorporates dimensions, measures, and filters.

cURL: 
curl -u myname:mypass https://api.enterprise.apigee.com/v1/o/{myorg}/environments/prod/stats/category?select=sum(price)&timeRange=1%2F1%2F2012%2000%3A00~2%2F10%2F2012%2000%3A00&timeUnit=minute&sortby=sum(price)&sort=DESC'
